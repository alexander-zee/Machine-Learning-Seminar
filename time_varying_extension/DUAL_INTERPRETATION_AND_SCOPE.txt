Dual Interpretation paper vs this codebase
==========================================

Reference: Goulet Coulombe, Göbel, Klieber — "Dual Interpretation of Machine
Learning Forecasts" (arXiv:2412.13076, Dec 2024).

What the paper does (conceptually)
----------------------------------
- Primal interpretation: explain predictions via contributions of predictors
  (features).
- Dual interpretation: the same prediction can be written as a weighted sum of
  *in-sample / training* values of the target, with weights that reflect
  *proximity* between the current economic state and past states.
- Those weights are likened to a "data portfolio"; the paper suggests
  diagnostics inspired by portfolios: concentration, turnover, short exposure,
  etc.
- Technically, the paper shows how to recover dual / proximity weights for
  specific estimators: (kernel) ridge, random forest, boosted trees, neural
  networks — using their dual solutions or auxiliary constructions.

What we implemented in time_varying_extension/ (aligned with the paper in spirit)
---------------------------------------------------------------------------------
- Historical analogue weights: Gaussian kernel on a low-dimensional *state*
  (value-weighted cross-sectional means of quantile-ranked LME and the two
  triplet characteristics), comparing month t to past months tau < t.
- Optional second factor: truncated geometric time kernel
  K_E(j) = lambda^j (1-lambda)/(1-lambda^m) 1{j < m}, j = t - tau (calendar age),
  multiplied with the Gaussian factor and renormalized — as in a two-kernel
  local weighting design.
- Dual-style diagnostics on those weights:
  * effective number of analogues = 1 / sum w_i^2 (concentration),
  * turnover of the kernel weight vector across months (TV distance),
  * plots: analogue concentration over time, MV weights over time, states vs a
    simple conditional premium proxy w'mu.
- One-triplet workflow, split-depth summaries for AP column names, top
  portfolio-pair co-movements — to match supervisor-style "interpret weights /
  splits / combinations".

What we did NOT implement from that paper
-----------------------------------------
- Closed-form dual weights for kernel ridge, RF, GBM, or neural nets (representer
  theorem route, auxiliary ridge on penultimate layer, etc.).
- Any change to the baseline Bryzgalova AP-tree + LASSO pipeline; the extension
  is a separate plug-in that reads existing CSV outputs.

How this relates to other readings (e.g. achievable AP-trees, transaction costs)
--------------------------------------------------------------------------------
- SSRN 5095345-style AAP-trees (net Sharpe, Bayesian optimization, smoothing psi)
  are a different extension; we do not replicate that here.

Commands to produce extension outputs (from repo root)
-------------------------------------------------------
Prerequisites:
  - data/prepared/panel.parquet   (run step1_prepare_data)
  - data/results/tree_portfolios/LME_OP_Investment/level_all_excess_combined_filtered.csv
  - Selected_Ports_10.csv under either
      data/results/grid_search/tree/LME_OP_Investment/
    or the same folder as the portfolio CSV

PowerShell (replace cd with your clone path):

  cd "C:\Users\Alexa\Downloads\Machine-Learning-Seminar-main (8)\Machine-Learning-Seminar-main"

  python -m time_varying_extension.run_example
  python -m time_varying_extension.run_example --time-decay
  python -m time_varying_extension.run_example --help

Optional: --out DIR --lambda-time 0.92 --m-window 96

Outputs: tv_mv_weights.csv, figures/*.png, interpretation_tv.txt,
tv_kernel_params.json, diagnostic CSVs under the output directory.
